# 教师端小车信息监控程序（Web）

## 项目概述
开发一个**教师端Web监控程序**，用于实时接收并可视化展示由小车（机器人）采集上报的现场信息。

## 交付物
- 一个完整的 Python 程序（包含后端服务与前端页面），部署在教师的笔记本电脑上运行。

## 输入数据格式
程序通过局域网（或本地）接收来自小车的 **JSON 格式消息**，每条消息包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `message_id` | 字符串/整数 | 消息唯一标识 |
| `axis_x` | 浮点数 | 小车当前位置 X 坐标 |
| `axis_y` | 浮点数 | 小车当前位置 Y 坐标 |
| `request` | 字符串 | 请求类型，枚举值：`"物品"` 或 `"教师协助"` |
| `description` | 字符串 | 当 `request` 为 `"物品"` 时，填写具体物品名称；当为 `"教师协助"` 时，填写请求协助的详细原因 |

## 功能需求

### 后端（Python）
- 开启一个网络服务端口（可配置），持续监听来自小车的 TCP/WebSocket 连接（建议使用 WebSocket 或 HTTP 长轮询）。
- 接收到 JSON 消息后，进行**合法性校验**（字段完整、类型正确）。
- 将解析后的有效消息**实时推送给前端**（可通过 WebSocket 或 SSE 方式）。

### 前端（Web界面）
- 以**卡片列表**或**表格**形式，按时间倒序展示所有接收到的消息。
- 突出显示 `request` 类型（“物品”与“教师协助”用不同颜色标签区分）。
- 在地图/坐标区域（可选）用散点图显示小车位置（`axis_x`, `axis_y`），点击点可查看详情。
- 页面需**自动刷新**，新消息到达后无需手动刷新即可更新显示。

## 非功能性要求
- **部署**：单机运行，后端与前端可同机部署，前端通过浏览器访问 `http://localhost:端口` 即可。
- **技术选型**：后端推荐使用 Flask + Flask-SocketIO 或 FastAPI + WebSocket；前端推荐使用原生 HTML/CSS/JavaScript，避免引入复杂框架。
- **可靠性**：程序应能处理连接异常、消息解析失败等情况，并记录日志，避免崩溃。
- **启动方式**：提供一键启动脚本（如 `run.py`），启动后自动打开浏览器访问前端页面。

## 验收标准
1. 后端能正常接收并解析示例 JSON 消息。
2. 前端页面能实时显示收到的消息，且不同请求类型有视觉区分。
3. 坐标信息能正确展示（至少以文本形式显示，最好有简易坐标图）。
4. 程序运行稳定，断开重连后能继续工作。
