### 需求
编写麦克风流式传输和识别代码，将手机麦克风音频流发送到电脑，由电脑进行音频识别并输出结果。

### 交付要求
一个 Python 程序，实现 [需求] 中的功能。

### 部署位置
代码部署在 Windows 笔记本电脑上。

### 音频传输方式
- 传输协议：WebRTC
- 使用的 Python 库：aiortc, aiohttp, numpy
- 虚拟局域网：Tailscale

### 参考程序
- `Baseline/offline_test` 目录下的 receiver.py

### 程序实现详细要求
1. 电脑端执行一个 Python 程序，创建一个网页，手机端打开该网页后向电脑端传输麦克风音频流。
2. 电脑端接收到麦克风音频流，将它传入 YAMNet 预训练模型识别哭声数据。
3. 电脑端将哭声数据识别结果输出。

---

按以下顺序依次解决问题：
1. 检测到一次哭声之后，改为间隔 5 秒（由一个宏定义）之后再执行下一次检测。
2. 手机端未同步“推理次数”“哭声事件”“哭声最高分”数据，将这些数据同步到网页。
3. 手机端点击“停止传输”后，再次点击“开始传输”无响应（需要刷新页面之后才可重新建立通信），修复这个问题。
