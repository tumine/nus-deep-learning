### 需求
编写带二重 PID 控制（直线轨迹控制、距离控制）的硬件层面直行函数，替换现有只具有直线轨迹控制 PID 硬件层面直行函数。

### 交付要求
一个修改之后的 `.ino` 程序，实现 [需求] 中的功能。

### 部署位置
代码部署在 Arduino 主板上。

### 硬件资源
- MPU-6500，可以输出当前测得的三轴加速度 `accelX/Y/Z` 和 三轴角速度 `gyroX/Y/Z`。
- MPU-6500 硬件连线：SCL 引脚连接到 21 口，SDA 引脚连接到 20 口。

### 可用的软件调用
- Arduino 库函数 `PID(&Input, &Output, &Setpoint, Kp, Ki, Kd, Direction)`，其中 `Setpoint` 为期望系统达到的目标值，`Direction` 为控制方向（当实际值低于设定值时系统的控制行为是增加输出，还是减少输出）
- Arduino 库函数 `Compute()`，调用效果是执行一次 PID 计算，更新 `Output` 的值。

### 参考程序
- `newrun.ino` 文件中的 `moveDistance` 函数，是带有 PID 控制直行轨迹的直行运动函数，但不能控制小车直行的距离。
- `newrun.ino` 文件中的现有 `turnAngle` 函数，是带 PID 控制程序的转向程序，其中传入参数 `angle_deg` 为正时，小车向左转向；传入参数 `angle_deg` 为负时，小车向右转向。

### 程序实现详细步骤
1. 利用 Arduino 库函数 `PID`（见 [可用的软件调用]），以及 MPU-6500 传出的各个运动状态参数（见 [硬件资源]），定义并传入直行距离 PID 控制算法需要的各个参数。
2. 在 `newrun.ino` 中，修改 `turnAngle` 函数实现，结合 `Compute()` 库函数（见 [可用的软件调用]），将直行函数改为带有 PID 控制的版本。

# 任务背景
在现有的 Arduino 小车控制程序（`newrun.ino`）中，现有的 `moveDistance` 函数仅具备基于 MPU-6500 陀螺仪数据的“直线航向偏航角 PID 控制”，无法实现对“直行行驶距离”的精准把控。
现在，我需要你升级该函数，利用 MPU-6500 的加速度数据进行二次积分或解算，实现**二重 PID 控制（直线航向控制 + 距离位置控制）**，替换原有的直行函数。

# 硬件资源与部署环境
- **主控板**：Arduino 主板。
- **传感器**：MPU-6500 六轴传感器。
  - 接口及连线：I2C 通信，SCL 引脚连接到 21 口，SDA 引脚连接到 20 口。
  - 数据输出：可读取当前三轴加速度 `accelX`, `accelY`, `accelZ` 和三轴角速度 `gyroX`, `gyroY`, `gyroZ`。

# 软件依赖与可用调用
- **PID 库对象定义**：`PID(&Input, &Output, &Setpoint, Kp, Ki, Kd, Direction)`
  - `Setpoint`：期望系统达到的目标值。
  - `Direction`：控制方向（`DIRECT` 或 `REVERSE`，即实际值低于设定值时，增加还是减少输出）。
- **PID 计算函数**：`Compute()`，调用后将基于最新反馈执行一次 PID 迭代，并更新 `Output` 的值。

# 参考程序说明
- **`moveDistance` 函数**（当前待修改目标）：当前已包含“直线轨迹控制”的 PID 逻辑，但缺乏距离环控制。
- **`turnAngle(angle_deg)` 函数**（仅供参考）：现有带 PID 的转向程序。当 `angle_deg > 0` 向左转，`angle_deg < 0` 向右转，可作为 PID 库在本项目中用法的参考。

# 程序实现详细步骤
请按照以下步骤编写并修改代码：
1. **定义距离环 PID 参数**：利用 Arduino 库函数 `PID` 定义直行距离控制所需的对象和变量（Input, Output, Setpoint, Kp, Ki, Kd 等）。
2. **运动学解算（关键）**：在直行循环中，精确获取时间差 `dt`，利用 MPU-6500 的加速度数据进行滤波、去零偏和积分运算，实时估算小车的已行驶距离。
3. **双环 PID 耦合控制**：
   - **距离环（外环）**：以目标距离为 `Setpoint`，估算距离为 `Input`，通过 `Compute()` 计算出基础前馈动力 `BasePWM`。
   - **偏航角环（内环）**：继续保留原有的直线轨迹控制逻辑，计算出左右轮的差速动力 `DiffPWM`。
   - 将两个环的输出结合，最终作用于电机控制引脚。
4. **修改函数实现**：重构 `newrun.ino` 中的 `moveDistance(float target_distance)` 函数，集成上述逻辑，并确保小车到达指定距离后能平稳刹停并退出控制循环。
