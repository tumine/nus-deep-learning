在暑期学校中参加 Deep Learning 小组，Deep Learning 小组与 Robotics 小组共同完成一个机器人小车项目，使用树莓派和小车、运用 Deep Learning 项目和 Robotics 项目的知识设计并实现一个具有简单应用价值的作品。

要求
1. 结合 PPT 分析，给出若干在实际生活、教育等场景下具有落地潜力的实现方向。
2. 在项目中至少使用两种模态。
3. 产品必须有可活动部件，但产品本身可以固定在一处。
4. 产品必须体现基于 Deep Learning 的自动化特征。
5. 产品必须具有实时的交互能力。

产品设计周期：11 天

团队构成：3 人来自 Deep Learning 小组，2 人来自 Robotics 小组

硬件局限性
1. 小车仅采用轮式计数器追踪轨迹，且外部无辅助定位设施，无法进行任何长距离精确运动或严格循迹运动。
2. 小车规格较小，不能增加过多配重。
3. 可使用的资源只包括基本的硬件设施，如摄像头、超声波传感器、LED 灯、小尺寸单色 OLED 屏幕等。

设计要求
1. 详细描述小车需要具备哪些基本模块，每个模块需要实现什么功能（假设例如，运动模块需要实现任意角度转弯）。
2. 描述在软件层面上，需要适配什么样的深度学习算法。具体而言，这些深度学习算法期望服务于什么功能需求。
