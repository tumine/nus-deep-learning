# Classroom Assistant Robot - ArUco Request Card Module

## Project Overview

This module is part of the **Classroom Assistant Robot** project.

It is responsible for recognizing **ArUco request cards** shown by children after the robot stops at their desk. Once a request card is confirmed, the module generates a delivery task and sends it to the robot control system.

The current supported classroom requests include:

| Marker ID | Request |
|-----------|---------|
| 0 | Building Blocks |
| 1 | Pencil |
| 2 | Eraser |
| 3 | Teacher Assistance |

The robot workflow is:

1. Patrol along the predefined route.
2. Detect a child raising a hand (implemented by another module).
3. Stop beside the child.
4. Wait for the child to display an ArUco request card.
5. Recognize the request.
6. Generate a delivery task.
7. Return to the teacher's station to collect the requested item.
8. Deliver the item to the child.
9. Resume patrol.

## Message Passing Interface

The system uses a structured message passing architecture to communicate detection events, tasks, and robot commands between modules. All messages are Python dictionaries (`dict`) with well-defined keys.

---

### 1. Hand Raise Event

Generated by `HandDetector.detect()` when a student raises their hand for several consecutive frames.

| Key | Type | Description |
|-----|------|-------------|
| `type` | `str` | Always `"hand_raise"`, identifies this as a hand detection event |
| `target` | `tuple(int, int)` | Center pixel coordinates `(x, y)` of the detected person's bounding box |
| `confidence` | `float` | YOLO detection confidence score (0.0 ~ 1.0) |
| `bbox` | `tuple(int, int, int, int)` | Bounding box of the person: `(x1, y1, x2, y2)` where `(x1, y1)` is top-left and `(x2, y2)` is bottom-right |
| `left_raised` | `bool` | `True` if left wrist is above left shoulder by the margin |
| `right_raised` | `bool` | `True` if right wrist is above right shoulder by the margin |

**Multi-frame confirmation**: A hand must be raised for `CONFIRM_FRAMES` (default: 5) consecutive frames before an event is emitted, preventing false positives.

**Example**:
```python
{
    "type": "hand_raise",
    "target": (320, 240),
    "confidence": 0.87,
    "bbox": (200, 100, 440, 400),
    "left_raised": False,
    "right_raised": True
}
```

---

### 2. Card Detection Result

Generated by `CardDetector.detect()` when an ArUco marker is detected and confirmed.

| Key | Type | Description |
|-----|------|-------------|
| `id` | `int` | ArUco marker ID (0 ~ 3) |
| `request` | `str` | Human-readable request name mapped from marker ID: `"blocks"`, `"pencil"`, `"eraser"`, or `"teacher"` |
| `center` | `tuple(int, int)` | Center pixel coordinates `(x, y)` of the detected marker |
| `corners` | `np.ndarray` | Four corner points of the marker as `(4, 2)` array of `int` |
| `count` | `int` | Number of consecutive frames the marker has been detected |
| `confirmed` | `bool` | `True` when the marker has been detected for `CONFIRM_FRAMES` frames, preventing duplicate firing |

**Multi-frame confirmation**: A marker must be detected for `CONFIRM_FRAMES` consecutive frames. Once confirmed, it will not trigger again until it leaves the frame.

**ArUco Marker Mapping**:
| Marker ID | `request` Value | Meaning |
|-----------|----------------|---------|
| 0 | `"blocks"` | Building Blocks |
| 1 | `"pencil"` | Pencil |
| 2 | `"eraser"` | Eraser |
| 3 | `"teacher"` | Teacher Assistance |

**Example**:
```python
{
    "id": 0,
    "request": "blocks",
    "center": (400, 300),
    "corners": array([[380, 280], [420, 280], [420, 320], [380, 320]]),
    "count": 5,
    "confirmed": True
}
```

---

### 3. Delivery Task

Generated by `RequestManager.create_task()` from a confirmed card detection result, then augmented with student context from the state machine.

| Key | Type | Description |
|-----|------|-------------|
| `type` | `str` | Always `"delivery"`, identifies this as a delivery task |
| `item` | `str` | The requested item name (same as `request` in card result): `"blocks"`, `"pencil"`, `"eraser"`, or `"teacher"` |
| `marker_id` | `int` | Original ArUco marker ID that triggered this task |
| `target` | `tuple(int, int)` | Center of the ArUco marker in image coordinates |
| `student_context` | `dict` | (Added by `main.py`) A copy of the state machine context at the time of the card event, containing student position and navigation info (see [State Machine Context](#6-state-machine-context)) |

**Example**:
```python
{
    "type": "delivery",
    "item": "blocks",
    "marker_id": 0,
    "target": (400, 300),
    "student_context": {
        "route_node": 1,
        "scan_direction": "left",
        "student_target": (320, 240),
        "student_confidence": 0.87,
        "approach_command": {
            "direction": "left",
            "target": (320, 240),
            "forward_seconds": 1.5
        }
    }
}
```

---

### 4. Robot Command (TCP Message)

Sent from `RobotController.execute()` to the Raspberry Pi robot control system via TCP socket as a JSON-encoded string. There are four command types, each corresponding to a state transition.

#### 4.1 `approach_student` вҖ” Navigate to the student who raised a hand

| Key | Type | Description |
|-----|------|-------------|
| `command` | `str` | Always `"approach_student"` |
| `route_node` | `int` or `None` | Current route node number the robot is at |
| `scan_direction` | `str` or `None` | Direction the robot was scanning: `"front"`, `"left"`, or `"right"` |
| `student_target` | `tuple(int, int)` or `None` | Center coordinates `(x, y)` of the student needing assistance |
| `approach` | `dict` or `None` | Approach movement details: `{"direction": str, "target": (x,y), "forward_seconds": float}` |

#### 4.2 `go_teacher` вҖ” Navigate back to the teacher station

| Key | Type | Description |
|-----|------|-------------|
| `command` | `str` | Always `"go_teacher"` |
| `task` | `dict` | The complete [Delivery Task](#3-delivery-task) containing the requested item and student context |
| `route_node` | `int` or `None` | Current route node the robot is at |

#### 4.3 `return_student` вҖ” Return to the student after collecting materials

| Key | Type | Description |
|-----|------|-------------|
| `command` | `str` | Always `"return_student"` |
| `task` | `dict` | The saved [Delivery Task](#3-delivery-task) (retrieved from `StateMachine.get_task()`) |
| `route_node` | `int` or `None` | Current route node |
| `scan_direction` | `str` or `None` | Direction to approach the student: `"front"`, `"left"`, or `"right"` |
| `student_target` | `tuple(int, int)` or `None` | Coordinates of the student |
| `approach` | `dict` or `None` | Approach movement parameters |

#### 4.4 `return_patrol` вҖ” Resume the patrol route

| Key | Type | Description |
|-----|------|-------------|
| `command` | `str` | Always `"return_patrol"` |
| `route_node` | `int` or `None` | Route node to return to |
| `scan_direction` | `str` or `None` | Original scanning direction to restore |
| `approach` | `dict` or `None` | Approach parameters for reference |

**TCP Communication**:
- **Protocol**: TCP (stream)
- **Host**: `100.84.2.68` (configurable in `RobotController`)
- **Port**: `2105` (configurable)
- **Format**: JSON string encoded as UTF-8 bytes
- **Timeout**: 2.0 seconds
- **Flow control**: `RobotController.busy` flag prevents sending a new command while the previous one is in flight

**Example** (approach_student):
```json
{
    "command": "approach_student",
    "route_node": 1,
    "scan_direction": "left",
    "student_target": [320, 240],
    "approach": {
        "direction": "left",
        "target": [320, 240],
        "forward_seconds": 1.5
    }
}
```

### 5. Person Info (Internal)

Generated internally by `HandDetector` for each detected person. Used by `HandDetector.draw()` for visualization.

| Key | Type | Description |
|-----|------|-------------|
| `bbox` | `tuple(int, int, int, int)` | Person bounding box `(x1, y1, x2, y2)` |
| `center` | `tuple(int, int)` | Center of the bounding box `(x, y)` |
| `confidence` | `float` | YOLO detection confidence (0.0 ~ 1.0) |
| `left_shoulder` | `tuple(int, int)` | COCO keypoint 5 вҖ” left shoulder pixel coordinates |
| `right_shoulder` | `tuple(int, int)` | COCO keypoint 6 вҖ” right shoulder pixel coordinates |
| `left_wrist` | `tuple(int, int)` | COCO keypoint 9 вҖ” left wrist pixel coordinates |
| `right_wrist` | `tuple(int, int)` | COCO keypoint 10 вҖ” right wrist pixel coordinates |
| `left_raised` | `bool` | Whether left wrist is above left shoulder |
| `right_raised` | `bool` | Whether right wrist is above right shoulder |
| `hand_raised` | `bool` | `True` if either hand is raised |

---

### 6. State Machine Context

Maintained by `StateMachine` throughout the robot's workflow. Stored as a dictionary and passed between components.

| Key | Type | Initial | Description |
|-----|------|---------|-------------|
| `route_node` | `int` or `None` | `None` | Current route/intersection node the robot is at |
| `scan_direction` | `str` or `None` | `None` | Direction the robot is scanning/facing: `"front"`, `"left"`, or `"right"` |
| `student_target` | `tuple` or `None` | `None` | Pixel coordinates of the student requiring service |
| `student_confidence` | `float` or `None` | `None` | Confidence score of the hand-raise detection |
| `approach_command` | `dict` or `None` | `None` | Movement parameters for approaching the student: `{"direction": str, "target": (x,y), "forward_seconds": float}` |

**Context methods**:
- `update_context(**kwargs)` вҖ” Update one or more context keys (raises `KeyError` for unknown keys)
- `get_context()` вҖ” Return a shallow copy of the full context dictionary
- `get_context_value(key, default)` вҖ” Get a single value with a fallback default
- `clear_context()` вҖ” Reset all context values to `None`

---

### 7. Marker State (Internal)

Maintained internally by `CardDetector` per detected ArUco marker.

| Key | Type | Description |
|-----|------|-------------|
| `count` | `int` | Number of consecutive frames this marker has been detected |
| `confirmed` | `bool` | `True` once the marker has been seen for `CONFIRM_FRAMES` frames |
